Orchestrator has been deployed to namespace {{ .Release.Namespace }}.

1. Verify the pods are ready:
   kubectl get pods -n {{ .Release.Namespace }} -l "app.kubernetes.io/name={{ include "orchestrator.name" . }}"

2. Check the readiness probe:
   kubectl port-forward svc/{{ include "orchestrator.fullname" . }} 8080:{{ .Values.service.port }} -n {{ .Release.Namespace }}
   curl http://localhost:8080/readyz

   Expected response: {"status":"ok","checks":{"forge":"ok","db":"ok","scheduler":"ok"}}

3. Configure the GitHub webhook:
   - URL:     https://{{ (first .Values.ingress.hosts).host }}/api/webhook
   - Content-Type: application/json
   - Secret:  The value of OPERATOR_SECRET_KEY in your {{ .Values.secretName }} secret
   - Events:  Issues, Pull Requests, Issue Comments, PR Review Comments

4. Log in (required before the authenticated API below and the web UI):
   Open https://{{ (first .Values.ingress.hosts).host }}/ and sign in as:
     username: admin
     password: <the OPERATOR_BOOTSTRAP_PASSWORD you set in {{ .Values.secretName }}>
   (The 'admin' operator is seeded at boot ONLY if OPERATOR_BOOTSTRAP_PASSWORD is set.
   Add operators via POST /api/operators once logged in. Note: operator accounts are
   in-memory until DB-backed stores land — admin re-seeds from the env on restart.)

5. Register the first repository (needs a bearer token from step 4):
   TOKEN=$(curl -s -X POST https://{{ (first .Values.ingress.hosts).host }}/api/auth \
     -H "Content-Type: application/json" \
     -d '{"username":"admin","password":"<OPERATOR_BOOTSTRAP_PASSWORD>"}' | jq -r .access_token)
   curl -X POST https://{{ (first .Values.ingress.hosts).host }}/api/repos \
     -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
     -d '{"owner":"<owner>","name":"<repo>","enabled":true,"intake_enabled":false}'

6. View pipeline status:
   https://{{ (first .Values.ingress.hosts).host }}/

Secret keys the operator must supply in {{ .Values.secretName }}:
  FORGE_TOKEN                  — GitHub PAT (PAT mode) or empty string (App mode)
  GITHUB_APP_ID                — GitHub App ID (App mode)
  GITHUB_APP_PRIVATE_KEY       — PEM-encoded RS256 private key (App mode)
  GITHUB_APP_INSTALLATION_ID   — Installation ID (App mode)
  OPERATOR_SECRET_KEY          — Webhook HMAC secret + operator JWT signing key (use >= 32
                                 bytes; HS256 emits a weak-key warning below that)
  OPERATOR_BOOTSTRAP_PASSWORD  — seeds the initial 'admin' operator at boot (REQUIRED to
                                 log in; without it no operator exists and login is impossible)
  CLAUDE_CODE_OAUTH_TOKEN      — Claude API OAuth token for agent harness
  PUSH_VAPID_PRIVATE_KEY       — VAPID private key for web push notifications

OCI Helm chart registry (issues #120, #89):
  Each release tag (e.g. v1.2.3) publishes a versioned chart artifact at:
    oci://ghcr.io/tuckermclean/charts/orchestrator:<semver>

  Chart.yaml in the repository keeps version/appVersion as "0.0.0-dev".
  The semver is stamped into the packaged artifact at release time only via
  `helm package --version <tag>` — never committed to the repo.

  Install directly from the OCI registry:
    helm upgrade --install orchestrator \
      oci://ghcr.io/tuckermclean/charts/orchestrator \
      --version <semver> -n orchestrator -f values-prod.yaml

  Flux GitOps consumers — use an OCIRepository pointing at the semver-tagged
  chart so Flux re-pulls chart templates (not just the container image) on every
  release.  Example Flux resources:

    apiVersion: source.toolkit.fluxcd.io/v1beta2
    kind: OCIRepository
    metadata:
      name: orchestrator-chart
      namespace: flux-system
    spec:
      interval: 5m
      url: oci://ghcr.io/tuckermclean/charts/orchestrator
      ref:
        semver: ">=1.0.0"   # or pin: tag: "1.2.3"
      provider: generic
    ---
    apiVersion: helm.toolkit.fluxcd.io/v2beta2
    kind: HelmRelease
    metadata:
      name: orchestrator
      namespace: orchestrator
    spec:
      interval: 10m
      chartRef:
        kind: OCIRepository
        name: orchestrator-chart
        namespace: flux-system
